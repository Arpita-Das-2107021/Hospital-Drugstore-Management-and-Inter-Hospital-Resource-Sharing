## How to Run

### Step 0 — Install dependencies
```bash
pip install numpy
# For real GPU training later:
# pip install torch torch-geometric
```

### Step 1 — Run everything
```bash
python run_all.py
```

Or run each step separately:
```bash
python data/generate_data.py       # generate random data
python utils/build_graph.py        # build healthcare graph
python models/train_stgnn.py       # train outbreak predictor
python models/marl_dispatcher.py   # run medicine dispatcher
```

### Neighbor nodes for routing and notifications

The runtime outbreak wrapper (`model2/pipeline.py`) exposes:

1. `run_outbreak_pipeline(...)` for standardized outbreak rows.
2. `run_outbreak_pipeline_with_neighbors(...)` for outbreak rows plus neighbor nodes.

Use the neighbor output for both of these decisions:

1. Request medicine from nearby nodes/depots first.
2. Notify neighboring nodes when a healthcare is outbreak-flagged.

Example shape:

```json
{
	"results": [{"healthcare_id": "PH001", "outbreak_probability": 0.82, "outbreak_flag": true}],
	"neighbors": {
		"PH001": [
			{"healthcare_id": "PH002", "upazila": "Akkelpur", "distance_km": 0.552}
		]
	}
}
```

Neighbor lists are sorted by shortest distance first. Use `max_neighbors` to limit fan-out.

## How to Use Real Data

### healthcares.csv
Replace the 20 random healthcares with your actual healthcares:
```
healthcare_id, name, upazila, lat, lon
PH001, Your Healthcare Name, Joypurhat Sadar, 24.9012, 89.0234
...
```

### sales.csv
One row per medicine sold per day per healthcare:
```
date, healthcare_id, medicine_name, quantity_sold, upazila
2024-01-01, PH001, Paracetamol 500mg, 45, Joypurhat Sadar
...
```

### medicines.csv
```
medicine_name, base_daily_sales, outbreak_multiplier, signals_disease
Paracetamol 500mg, 30, 3.5, Fever/Flu
...
```
`signals_disease` must be one of: Fever/Flu | Diarrhea | Respiratory | Allergy/Fever | Normal

## What Each Paper Contributes

| Paper | What it adds to your system |
|-------|----------------------------|
| Paper 1 (VRE/Hospital GNN) | Heterogeneous graph structure, "living map" concept, temporal daily snapshots |
| Paper 2 (Evolve-DGN MARL) | Evolving GNN for changing graphs, MARL agents for resource dispatch, equity reward |
| Paper 3 (DGO-ST-GNN) | Spatiotemporal prediction from sales data, Grasshopper optimization, 97% accuracy target |
